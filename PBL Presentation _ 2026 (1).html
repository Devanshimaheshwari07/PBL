A Lightweight Hybrid Model for Detecting AI-Generated Text Using Statistical and Linguistic Features

Problem Statement

With the rapid advancement of large language models such as GPT-4, LLaMA, and Gemini, distinguishing between human-written and AI-generated text has become increasingly difficult. This creates serious concerns in academic integrity, journalism, and digital trust.

Existing detection approaches suffer from major limitations:

  • Deep learning models achieve high accuracy but require high computational resources.
  • Statistical and linguistic methods are efficient but lack strong performance individually.
  • Most systems lack interpretability, making decisions difficult to justify.

Therefore, there is a need for a lightweight, accurate, and interpretable system that can classify AI-generated text efficiently and be deployed in real-world environments.

Objectives

  • Develop a hybrid AI-text detection model
  • Combine statistical and linguistic features
  • Improve accuracy without deep learning
  • Ensure interpretability
  • Enable real-world deployment

Research Gap / Innovation

  • Deep learning models are resource-intensive
  • Single-feature models lack accuracy
  • Limited interpretability in existing systems
  • Poor generalization across datasets

Innovation

  • High accuracy (91.6%) without GPU
  • Feature importance analysis included
  • Lightweight and deployable system

Literature Review

The problem of detecting AI-generated text has gained significant attention with the rise of advanced large language models. Existing approaches can be broadly categorized into statistical, linguistic, and deep learning-based methods.

Statistical and stylometric approaches focus on analyzing writing patterns such as word frequency, sentence length, entropy, and lexical diversity. Tools like GLTR utilize probability distributions of words generated by language models to identify machine-generated text. These approaches are computationally efficient and interpretable but struggle with modern AI models that generate more human-like text.

Linguistic and psycholinguistic approaches analyze deeper structural aspects of language, including part-of-speech (POS) distributions, syntactic complexity, parse tree depth, and discourse cohesion. Research shows that AI-generated text tends to have more uniform sentence structures and lower syntactic variability compared to human-written text. While these methods provide meaningful insights, they are not sufficient alone for high accuracy.

Deep learning-based approaches, particularly transformer models such as RoBERTa and DetectGPT, have demonstrated high accuracy in detecting AI-generated text. These models leverage contextual embeddings and probability-based scoring. However, they require large computational resources, GPU support, and are often difficult to interpret. Additionally, their performance may degrade when exposed to newer or unseen AI models.

Recent research explores hybrid approaches that combine multiple feature types to improve performance. By integrating statistical and linguistic features, these models aim to balance accuracy, efficiency, and interpretability. However, there is limited work focusing on lightweight hybrid systems that are both practical and deployable, which motivates the proposed approach.

Methodology

Feature Extraction

  • Statistical: TTR, entropy, punctuation density, compression ratio
  • Linguistic: POS distribution, parse depth, MTLD, cohesion index

Feature Fusion

  • Normalization
  • Concatenation
  • PCA

Models

  • Logistic Regression
  • Random Forest (Final Model)

Workflow

  1. Input text
  2. Extract features
  3. Apply PCA
  4. Classify using Random Forest

Dataset

  • HC3 Dataset
  • GPT-Wiki-Intro Dataset
  • SemEval-2024 Task 8

Total: 12,000 balanced samples

Results

The proposed hybrid model demonstrates strong performance across multiple benchmark datasets. The Random Forest classifier trained on combined statistical and linguistic features achieved an overall accuracy of 91.6%, significantly outperforming models trained on individual feature sets.

When compared to baseline approaches, the statistical-only model achieved an accuracy of 85.7%, while the linguistic-only model achieved 83.9%. The hybrid model improved performance by effectively combining complementary information from both feature types.

The model also achieved balanced precision (92.1%) and recall (91.2%), indicating that it performs well in both identifying AI-generated text and minimizing false classifications. This balance is crucial in real-world applications such as academic integrity systems.

Feature importance analysis revealed that compression ratio, type-token ratio, and average sentence length are the most influential features in distinguishing AI-generated text. Among linguistic features, parse tree depth and discourse cohesion also contributed significantly.

In terms of efficiency, the model is highly lightweight and does not require GPU support. It processes approximately 1000 samples in around 4 seconds on a standard CPU, making it suitable for deployment in resource-constrained environments such as educational platforms and editorial systems.
